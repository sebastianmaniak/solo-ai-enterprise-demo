Usage Guide

How to interact with the Solo Bank AI agents, explore traces, and query the wiki API.

Getting Started

The Solo Bank demo runs entirely inside a local Kind cluster named solo-bank-demo. Some services are exposed via NodePort (no port-forward needed), and others require kubectl port-forward to access from your browser or terminal.

Direct Access (NodePort — no port-forward needed)

Service URL Description
AgentRegistry http://localhost:30121 Skill catalog REST API
Docs Site http://localhost:30500 This documentation site

Port-Forward Commands (Management UI, kagent API, and Bank Wiki)

Run each command in a separate terminal and keep them running while you use the UI:

# Management UI — kagent management dashboard
kubectl port-forward svc/solo-enterprise-ui -n kagent 8090:8090

# Then open: http://localhost:8090

# kagent API — agent runtime API
kubectl port-forward svc/kagent-controller -n kagent 8083:8083

# Bank Wiki Server — direct API access
kubectl port-forward svc/bank-wiki-server -n bank-wiki 8080:8080

# Then open: http://localhost:8080/health
ℹ️
Kind Cluster Required The demo requires the Kind cluster solo-bank-demo to be running. If you haven't set it up yet, run ./setup.sh from the project root. You'll need OPENAI_API_KEY and ANTHROPIC_API_KEY set in your environment.

Talking to the Triage Agent

The Triage Agent (bank-triage-agent) is the front door to the Solo Bank system. Open the Management UI at http://localhost:8090, select bank-triage-agent from the agent dropdown, and type your request.

The Triage Agent does not answer banking questions itself — it greets you, identifies your need, and routes you to the right specialist with a brief explanation.

Example Prompts

I want to check my account balance
What mortgage rate can I get for a 30-year fixed loan?
I need help with a suspicious transaction on my account
I'm interested in refinancing my home
Can you help me find out if I qualify for a home equity line?

Expected Routing Behavior

Request Type Routes To Notes
Account balance, transactions, disputes Customer Service Agent Will ask for identity verification
Mortgage rates, refinancing, home equity Mortgage Advisor Agent Will look up credit profile
Suspicious activity, fraud report Customer Service Agent Compliance is internal only; CS handles escalation
Ambiguous requests Clarifying question Asks exactly one question before routing

Using the Customer Service Agent

The Customer Service Agent (bank-customer-service-agent) handles account inquiries, balance lookups, and transaction reviews. It is powered by GPT-4o Mini and has access to the bank-customer-tools and bank-transaction-tools MCP servers.

⚠️
Identity Verification Required Before sharing any account details, the Customer Service Agent will always ask for the customer's full name or Customer ID. This step cannot be skipped — it's part of the system prompt's security instructions.

Example Conversation Flow

You:    I need to check my account balance.

Agent:  Of course! To protect your privacy, I'll need to verify
        your identity first. Could you please provide your full
        name or your Customer ID?

You:    I'm John Smith.

Agent:  Thank you, John. I found your profile. You have two accounts:
        - Checking Account (****1234): $4,280.50
        - Savings Account (****5678): $12,450.00

        Would you like to review recent transactions on either account?

You:    Yes, show me the last 5 transactions on my checking account.

Agent:  Here are your 5 most recent checking account transactions:
        1. 2025-01-15  Whole Foods Market         -$87.43
        2. 2025-01-14  Direct Deposit — Payroll   +$3,200.00
        3. 2025-01-13  Netflix                    -$15.99
        4. 2025-01-12  Shell Gas Station          -$62.10
        5. 2025-01-11  Amazon.com                 -$129.99

Using the Mortgage Advisor Agent

The Mortgage Advisor Agent (bank-mortgage-advisor-agent) provides personalized mortgage rate quotes by looking up the customer's credit score, applying bank rate tables, and checking lending policies. Powered by Claude Sonnet.

How Personalized Rates Work

The agent follows a structured advisory process:

  1. Looks up the customer profile to retrieve credit score and salary
  2. Determines the credit tier using the credit-score-tiers policy
  3. Retrieves current rates for the requested mortgage type from the rate table
  4. Calculates Debt-to-Income (DTI) ratio using salary and existing obligations
  5. Checks down payment requirements and relationship discount eligibility
  6. Presents a clear, policy-cited recommendation

Example Prompt

What rate would customer CUST-00042 qualify for on a 30-year fixed mortgage?

Sample Response Format

Based on the customer profile for CUST-00042 (Maria Garcia):

Credit Score:  742 → Credit Tier: "Good" (720–759)
Annual Salary: $95,000
Existing Debt: $850/month

Current 30-Year Fixed Rates (per policy "mortgage-rates-2025"):
  Good tier rate: 6.75% APR

DTI Calculation:
  Monthly Income: $7,917
  Proposed payment (for $350,000 loan at 6.75%): $2,270/month
  Total DTI: (850 + 2,270) / 7,917 = 39.4% — within the 43% policy limit

Customer qualifies for a 30-year fixed at 6.75% APR.
Maria has been a customer for 6 years, making her eligible for
the 5-year relationship discount: -0.125%, bringing her to 6.625%.

Recommendation: Approve at 6.625% with standard 20% down payment.

More Prompts to Try

Can CUST-00003 qualify for a 15-year fixed mortgage?
What would refinancing look like for a customer with a 680 credit score?
What's the minimum down payment for a jumbo loan?
Which customers would benefit most from refinancing at current rates?

Using the Compliance Agent

The Compliance Agent (bank-compliance-agent) is an internal-only agent for bank staff. It reviews customer accounts for AML/KYC flags, suspicious transaction patterns, rate mismatches, and regulatory violations. Never route customers to this agent.

It has full access to all three MCP servers: customer tools, policy tools, and transaction tools. Powered by Claude Sonnet.

Example Prompt

Review customer CUST-00003 for compliance issues

Sample Structured Output

## Compliance Review: James Wilson (CUST-00003)
### Risk Rating: Medium

### Findings:
- **AML-001**: Transaction on 2025-01-08 of $11,200 (wire transfer, international)
  exceeds $10,000 threshold — per KYC/AML Policy §4.2, must have SAR filed within 30 days.
  No SAR record found in transaction notes.

- **RATE-002**: Customer's credit card APR is 22.99%. Credit score is 715 (Fair tier).
  Policy "credit-card-apr-schedule" specifies Fair tier max APR of 21.99%.
  Customer is being charged 1.00% above policy ceiling.

- **KYC-001**: Customer has an active AML flag (set 2025-01-08) with no escalation record.
  KYC Policy §2.1 requires escalation to compliance officer within 5 business days.

### Recommendations:
- File SAR for the $11,200 international wire transfer immediately
- Correct credit card APR from 22.99% to 21.99% in the next billing cycle
- Escalate AML flag to senior compliance officer and document in CRM
- Schedule 90-day enhanced monitoring per KYC Policy §3.4

More Compliance Prompts

Review customer CUST-00001 for AML red flags in the last 90 days
Are all customers with active flags properly escalated?
Check whether CUST-00042 is receiving the correct mortgage rate for their credit tier
Run a compliance audit across all customers with international wire transfers

Using the Management UI

The Management UI (solo-enterprise-ui) provides a dashboard for managing kagent agents, viewing agent configurations, and interacting with agents directly.

How to Access the Management UI

# Start the port-forward (keep this terminal open)
kubectl port-forward svc/solo-enterprise-ui -n kagent 8090:8090

# Open in browser
open http://localhost:8090

What the Management UI Provides

  • Agent management — view and configure Agent CRDs
  • Chat interface — interact with agents directly in the browser
  • Tool configuration — view RemoteMCPServer connections and tool availability
  • Model configuration — view ModelConfig CRDs pointing to LLM APIs
💡
Tip: Select an Agent Use the agent dropdown in the Management UI to switch between agents. Each agent's available tools are determined by the RemoteMCPServer references in its Agent CRD.

Exploring the Wiki Directly

The Bank Wiki server hosts all policy documents, rate tables, and KYC/AML guidelines referenced by the Mortgage Advisor and Compliance agents. You can query it directly using kubectl exec or via port-forward.

Health Check

WIKI_POD=$(kubectl get pod -l app=bank-wiki-server -n bank-wiki \
  -o jsonpath='{.items[0].metadata.name}')

kubectl exec "$WIKI_POD" -n bank-wiki -- wget -qO- http://localhost:8080/health

List All Documents

kubectl exec "$WIKI_POD" -n bank-wiki -- \
  wget -qO- http://localhost:8080/api/documents | jq '.documents[].title'

Search the Wiki

# Search for AML-related policies
kubectl exec "$WIKI_POD" -n bank-wiki -- \
  wget -qO- "http://localhost:8080/api/search?q=AML" | jq '.'

# Search for mortgage rate tables
kubectl exec "$WIKI_POD" -n bank-wiki -- \
  wget -qO- "http://localhost:8080/api/search?q=mortgage+rates" | jq '.'

Port-Forward Alternative

# Port-forward the wiki server
kubectl port-forward svc/bank-wiki-server -n bank-wiki 8080:8080

# Then use curl from your local machine
curl http://localhost:8080/api/search?q=credit-score-tiers
curl http://localhost:8080/api/documents/lending-policy-2025

Available Wiki Endpoints

Method Path Description
GET /health Health check — returns 200 OK when ready
GET /api/documents List all policy and rate documents
GET /api/documents/:slug Retrieve a specific document by slug
GET /api/search?q=query Full-text search across all documents
