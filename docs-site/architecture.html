Request Data Flow

The numbered steps below trace a complete request from a user typing in the Management UI to a tool response returning through the stack.

1

User Sends a Prompt

The user opens the Management UI (http://localhost:30090), selects an agent (e.g., bank-customer-service-agent), and types a message such as "I'm John Smith, what's my balance?"

2

kagent Invokes the LLM

The kagent runtime reads the Agent CRD for bank-customer-service-agent, assembles the system prompt and conversation history, and makes a direct API call to the configured LLM via its ModelConfig CRD (e.g., openai-gpt4o-mini pointing to the OpenAI API with an API key secret). The LLM responds with a tool call: get_customer_by_name({"name": "John Smith"}).

3

KMCP Routes the MCP Tool Call

kagent looks up the RemoteMCPServer named bank-customer-tools and KMCP sends the MCP tools/call request directly to the tool server in the bank-wiki namespace. KMCP handles the MCP protocol routing between the kagent controller and the remote MCP tool servers.

4

MCP Tool Server Executes the Tool

The bank-customer-tools pod in namespace: bank-wiki receives the MCP request, executes the get_customer_by_name function against its in-memory customer database, and returns a JSON result.

5

LLM Generates Final Response

The tool result is returned via KMCP to kagent, which feeds it back to the LLM. The LLM composes a final human-readable response: "Thank you, John. Your checking account balance is $4,280.50 and your savings balance is $12,450.00." This response is streamed back to the Management UI.

MCP Tool Inventory

Each MCP server exposes multiple tools. Below is a summary of what each server provides:

bank-customer-tools (port 8081)

Tool Name Description
get_customer_by_id Look up a customer profile by Customer ID (e.g., CUST-00042)
get_customer_by_name Search for a customer by full name or partial name
get_account_balance Retrieve account balances for all accounts associated with a customer
search_customers Search customers by various criteria (city, credit tier, account type)
list_customers Return a paginated list of all customers

bank-policy-tools (port 8082)

Tool Name Description
get_policy Retrieve a specific bank policy document by slug (e.g., "lending-policy-2025")
list_policies List all available bank policy documents
get_rate_table Retrieve current rate tables for mortgage products by credit tier
search_policies Full-text search across all policy documents
get_credit_tier Determine the credit tier (Excellent/Good/Fair/Poor) for a given credit score

bank-transaction-tools (port 8083)

Tool Name Description
get_transactions Retrieve recent transactions for a customer, with optional date range and limit
get_transaction_by_id Look up a specific transaction by its transaction ID
search_transactions Search transactions by amount, merchant, date, or type
get_large_transactions Return transactions above a specified amount threshold (AML use case)
get_account_summary Return aggregate account stats: total debits, credits, and balance trend

kagent CRD Configuration

kagent Enterprise uses Kubernetes Custom Resource Definitions (CRDs) to define agents, model connections, and MCP tool server references. The core CRD types are:

CRD Kind Purpose Example
Agent Defines an agent with system prompt, model reference, and tool bindings bank-triage-agent, bank-customer-service-agent
ModelConfig Points to an LLM API (OpenAI, Anthropic) with API key secret reference openai-gpt4o-mini, anthropic-claude-sonnet
RemoteMCPServer Declares the URL and protocol for an MCP tool server accessible via KMCP bank-customer-tools, bank-policy-tools
