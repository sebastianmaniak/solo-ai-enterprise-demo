Build Your Own Agent

A step-by-step tutorial: create a custom MCP tool server in Go, register it in AgentRegistry, wire it to a kagent Agent CRD, and test it — all in about 30 minutes.

Overview

In this tutorial you'll build an Account Summary Agent — a custom agent that generates brief, at-a-glance summaries of customer accounts. You'll learn every layer of the Solo.io AI Enterprise Platform:

  • Writing an MCP tool server in Go using the MCP SDK
  • Packaging it as a Docker image and loading it into Kind
  • Registering the skill in AgentRegistry via REST API
  • Creating a RemoteMCPServer Kubernetes resource
  • Defining an Agent CRD and wiring it to your tool server
  • Testing the agent live in the kagent UI
ℹ️
Prerequisites You should have completed the initial ./setup.sh and have the solo-bank-demo Kind cluster running. You'll need Go 1.21+, Docker, and kubectl configured for the cluster.

Step-by-Step Tutorial

1

Create an MCP Tool Server in Go

MCP (Model Context Protocol) tool servers expose callable functions to AI agents over HTTP. Create the directory structure and write the main server file: 

mkdir -p sample-agent/mcp-server
cd sample-agent/mcp-server
go mod init account-summary-tools

Then create sample-agent/mcp-server/main.go:

package main

import (
    "context"
    "encoding/json"
    "fmt"
    "log"
    "net/http"
    "os"

    "github.com/mark3labs/mcp-go/mcp"
    "github.com/mark3labs/mcp-go/server"
)

// AccountSummary holds the data we return to the agent
type AccountSummary struct {
    CustomerID   string  `json:"customer_id"`
    Name         string  `json:"name"`
    CreditScore  int     `json:"credit_score"`
    TotalBalance float64 `json:"total_balance"`
    AccountCount int     `json:"account_count"`
    RiskLevel    string  `json:"risk_level"`
    Summary      string  `json:"summary"`
}

func main() {
    // Create the MCP server
    s := server.NewMCPServer(
        "account-summary-tools",
        "1.0.0",
        server.WithToolCapabilities(true),
    )

    // Register the get_account_summary tool
    tool := mcp.NewTool(
        "get_account_summary",
        mcp.WithDescription(
            "Generate a brief account summary for a bank customer. "+
            "Returns credit score, total balance, account count, "+
            "risk level, and a plain-language summary.",
        ),
        mcp.WithString(
            "customer_id",
            mcp.Required(),
            mcp.Description("The customer ID, e.g. CUST-00042"),
        ),
    )

    s.AddTool(tool, func(ctx context.Context, req mcp.CallToolRequest) (*mcp.CallToolResult, error) {
        customerID, ok := req.Params.Arguments["customer_id"].(string)
        if !ok || customerID == "" {
            return mcp.NewToolResultError("customer_id is required"), nil
        }

        // In a real implementation, call the customer-tools MCP server.
        // Here we return mock data for the tutorial.
        summary := generateSummary(customerID)

        data, err := json.MarshalIndent(summary, "", "  ")
        if err != nil {
            return mcp.NewToolResultError(fmt.Sprintf("marshal error: %v", err)), nil
        }

        return mcp.NewToolResultText(string(data)), nil
    })

    // Start the HTTP server
    port := os.Getenv("PORT")
    if port == "" {
        port = "8084"
    }

    httpServer := server.NewStreamableHTTPServer(s)
    log.Printf("Account Summary Tools MCP server listening on :%s/mcp", port)
    if err := http.ListenAndServe(":"+port, httpServer); err != nil {
        log.Fatalf("Server error: %v", err)
    }
}

// generateSummary returns a mock account summary.
// In production, replace this with calls to bank-customer-tools.
func generateSummary(customerID string) AccountSummary {
    // Simulate different customers
    summaries := map[string]AccountSummary{
        "CUST-00042": {
            CustomerID:   "CUST-00042",
            Name:         "Maria Garcia",
            CreditScore:  742,
            TotalBalance: 28450.00,
            AccountCount: 3,
            RiskLevel:    "Low",
            Summary: "Maria is a long-standing customer (6 years) with good credit " +
                "and healthy balances across checking, savings, and a CD account. " +
                "No flags or compliance concerns.",
        },
        "CUST-00003": {
            CustomerID:   "CUST-00003",
            Name:         "James Wilson",
            CreditScore:  715,
            TotalBalance: 6200.00,
            AccountCount: 2,
            RiskLevel:    "Medium",
            Summary: "James has fair credit and moderate balances. Active AML flag " +
                "from an international wire transfer on 2025-01-08 requires follow-up.",
        },
    }

    if s, ok := summaries[customerID]; ok {
        return s
    }

    return AccountSummary{
        CustomerID:   customerID,
        Name:         "Unknown",
        CreditScore:  0,
        TotalBalance: 0,
        AccountCount: 0,
        RiskLevel:    "Unknown",
        Summary:      fmt.Sprintf("No data found for customer %s.", customerID),
    }
}

Add the MCP SDK dependency:

go get github.com/mark3labs/mcp-go@latest
go mod tidy
2

Build the Docker Image and Load into Kind

Create a Dockerfile for the MCP server, build the image, and load it directly into the Kind cluster (no registry required):

Create sample-agent/mcp-server/Dockerfile:

FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o /account-summary-tools .

FROM scratch
COPY --from=builder /account-summary-tools /account-summary-tools
EXPOSE 8084
ENTRYPOINT ["/account-summary-tools"]

Build and load into the cluster:

# Build the image
docker build -t account-summary-tools:latest sample-agent/mcp-server/

# Load directly into Kind — no registry needed
kind load docker-image account-summary-tools:latest --name solo-bank-demo

# Verify the image is available in the cluster
docker exec solo-bank-demo-control-plane crictl images | grep account-summary

Create the Kubernetes deployment at sample-agent/manifests/deployment.yaml:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: account-summary-tools
  namespace: bank-wiki
  labels:
    app: account-summary-tools
spec:
  replicas: 1
  selector:
    matchLabels:
      app: account-summary-tools
  template:
    metadata:
      labels:
        app: account-summary-tools
    spec:
      containers:
      - name: account-summary-tools
        image: account-summary-tools:latest
        imagePullPolicy: Never
        ports:
        - containerPort: 8084
        env:
        - name: PORT
          value: "8084"
---
apiVersion: v1
kind: Service
metadata:
  name: account-summary-tools
  namespace: bank-wiki
spec:
  selector:
    app: account-summary-tools
  ports:
  - port: 8084
    targetPort: 8084

Apply the deployment:

kubectl apply -f sample-agent/manifests/deployment.yaml

# Wait for the pod to be ready
kubectl wait --for=condition=ready pod -l app=account-summary-tools \
  -n bank-wiki --timeout=60s

# Verify the MCP server is responding
TOOL_POD=$(kubectl get pod -l app=account-summary-tools -n bank-wiki \
  -o jsonpath='{.items[0].metadata.name}')

kubectl exec "$TOOL_POD" -n bank-wiki -- wget -qO- http://localhost:8084/mcp

Register the Skill in AgentRegistry

AgentRegistry is a catalog of skills and tools. Register your new tool server so it can be discovered by other agents and tooling: 

curl -X POST http://localhost:30121/api/v1/skills \
  -H "Content-Type: application/json" \
  -d '{
    "name": "account-summary",
    "description": "Generate brief customer account summaries with credit score, balance, and risk level",
    "version": "1.0.0",
    "tags": ["banking", "accounts", "summary"],
    "tools": [
      {
        "name": "get_account_summary",
        "description": "Get a brief account summary for a customer by their Customer ID",
        "parameters": {
          "customer_id": {
            "type": "string",
            "description": "The customer ID, e.g. CUST-00042",
            "required": true
          }
        }
      }
    ]
  }'

Verify the skill was registered:

# List all registered skills
curl http://localhost:30121/api/v1/skills | jq '.skills[].name'

# Fetch the specific skill
curl http://localhost:30121/api/v1/skills/account-summary | jq '.'
💡
AgentRegistry NodePort AgentRegistry is exposed on NodePort 30121 by default in the demo cluster. If you're accessing it differently (e.g., from inside the cluster), use the service DNS: http://agentregistry.agentregistry.svc.cluster.local:80
4

Create the RemoteMCPServer Resource

A RemoteMCPServer is a kagent CRD that tells the agent runtime where your MCP tool server lives. Create it at sample-agent/manifests/remote-mcp-server.yaml: 

apiVersion: kagent.dev/v1alpha2
kind: RemoteMCPServer
metadata:
  name: account-summary-tools
  namespace: kagent
spec:
  url: http://account-summary-tools.bank-wiki.svc.cluster.local:8084/mcp
  protocol: STREAMABLE_HTTP
  description: "Account summary tools — generates brief customer account summaries"

Apply the resource and verify:

kubectl apply -f sample-agent/manifests/remote-mcp-server.yaml

# Check the RemoteMCPServer status
kubectl get remotemcpserver -n kagent account-summary-tools -o yaml

# kagent will probe the /mcp endpoint — check that it shows Ready
kubectl describe remotemcpserver -n kagent account-summary-tools

Create the Agent CRD

Now define the agent itself. Create sample-agent/manifests/agent.yaml with a system prompt that instructs the agent how to use your tool: 

apiVersion: kagent.dev/v1alpha2
kind: Agent
metadata:
  name: bank-account-summary-agent
  namespace: kagent
spec:
  description: "Solo Bank — Account Summary Agent. Generates at-a-glance summaries for bank staff."
  type: Declarative
  declarative:
    modelConfig: anthropic-claude-sonnet
    systemMessage: |
      You are an account summary assistant at Solo Bank. You help bank staff quickly
      understand a customer's overall financial standing with a brief, readable summary.

      When given a customer ID or name:
      1. Use the get_account_summary tool to retrieve their summary data
      2. Present the key facts in a clear, structured format
      3. Highlight any risk indicators or flags that require attention
      4. Keep the response concise — this is meant for a quick briefing, not a full review

      Format your response as:
      **Customer:** [Name] ([ID])
      **Credit Score:** [score] | **Risk Level:** [level]
      **Total Balance:** $[amount] across [N] accounts
      **Summary:** [plain-language summary]
      **Action Required:** [any flags or next steps, or "None"]

      If the customer is not found, say so clearly and suggest verifying the Customer ID.
    tools:
    - type: McpServer
      mcpServer:
        name: account-summary-tools
        kind: RemoteMCPServer

Apply the agent:

kubectl apply -f sample-agent/manifests/agent.yaml

# Verify the agent was created
kubectl get agents -n kagent bank-account-summary-agent

# Check the agent's status and tool wiring
kubectl describe agent -n kagent bank-account-summary-agent

Test Your Agent

Open the kagent UI, select your new agent, and try it out: 

# Make sure the kagent UI port-forward is running
kubectl port-forward svc/kagent-enterprise -n kagent 3000:80

# Open in browser
open http://localhost:3000

In the kagent UI, select bank-account-summary-agent from the agent dropdown and try these prompts:

Give me a summary for Maria Garcia
Summarize account CUST-00042
What's the overview for customer CUST-00003?

You can also test the tool server directly without going through the agent:

# Direct MCP tool call via kubectl exec
TOOL_POD=$(kubectl get pod -l app=account-summary-tools -n bank-wiki \
  -o jsonpath='{.items[0].metadata.name}')

kubectl exec "$TOOL_POD" -n bank-wiki -- wget -qO- \
  --post-data='{"jsonrpc":"2.0","method":"tools/call","params":{"name":"get_account_summary","arguments":{"customer_id":"CUST-00042"}},"id":1}' \
  --header='Content-Type: application/json' \
  http://localhost:8084/mcp | jq '.'

What's Next

Congratulations — you've built and deployed a custom agent on the Solo.io AI Enterprise Platform. Here are some ways to extend what you've built:

Connect to Real Data

Replace the mock data in generateSummary() with real HTTP calls to the bank-customer-tools MCP server. Use the service DNS: bank-customer-tools.bank-wiki.svc.cluster.local:8081/mcp

Add More Tools

Register additional functions on the same MCP server — for example, get_recent_alerts or get_product_recommendations. Each tool becomes callable by any agent that references your RemoteMCPServer.

Observe with AgentGateway

Route your agent's tool calls through AgentGateway to get OTEL tracing, rate limiting, and policy enforcement on every tool invocation. Update the RemoteMCPServer URL to point to the gateway.
