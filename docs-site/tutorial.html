See what the script does

The create.sh script runs 6 steps in order:

  1. Build — Compiles a Go MCP tool server into a Docker image
  2. Load — Pushes the image into the Kind cluster (no registry needed)
  3. Deploy — Creates a Deployment + Service in the bank-wiki namespace
  4. RemoteMCPServer — Creates a CRD so kagent discovers the tool via MCP protocol
  5. Agent — Creates an Agent CRD with system prompt and tool reference
  6. Register — Adds the agent and skill to the AgentRegistry catalog
See the MCP server code (Go)

The tool server connects to the bank wiki to fetch real customer data:

sample-agent/mcp-server/main.go (key parts) 
// Tool handler — fetches customer page from the wiki, extracts summary
func handleGetAccountSummary(ctx context.Context, req *mcp.CallToolRequest,
    args GetAccountSummaryArgs) (*mcp.CallToolResult, any, error) {

    namePath := strings.ToLower(strings.ReplaceAll(args.CustomerName, " ", "-"))
    pageURL := fmt.Sprintf("%s/wiki/customers/%s", wikiBaseURL, namePath)

    resp, err := httpClient.Get(pageURL)
    // ... parse wiki markdown, extract credit score, balances, flags ...

    return &mcp.CallToolResult{
        Content: []mcp.Content{&mcp.TextContent{Text: summary}},
    }, nil, nil
}

func main() {
    server := mcp.NewServer(&mcp.Implementation{
        Name: "account-summary-tools", Version: "1.0.0",
    }, nil)

    mcp.AddTool(server, &mcp.Tool{
        Name:        "get_account_summary",
        Description: "Get a brief account summary for a Solo Bank customer.",
    }, handleGetAccountSummary)

    handler := mcp.NewStreamableHTTPHandler(
        func(r *http.Request) *mcp.Server { return server }, nil)

    mux := http.NewServeMux()
    mux.Handle("/mcp", handler)
    mux.Handle("/mcp/", handler)
    http.ListenAndServe(":8085", mux)
}

What gets removed
  • account-summary-agent Agent CRD from kagent namespace
